Development Approach

  • Schema-aware RAG pipeline design with SELECT-only validation
  • Comparative evaluation of 5 LLM configurations (different models and embeddings)
  • Iterative testing on benchmark queries for accuracy assessment
  • Security-first implementation with input validation and SQL injection prevention
  • 100% local deployment architecture to ensure data privacy

Dataset & Hardware

Test Database: E-commerce database with customer, order, and product tables
Benchmark Queries: 5 standard queries covering aggregations, joins, and filtering
Model Configurations: 5 combinations tested (Llama 3.1:8b, Qwen3:1.7b, Gemma3:1b with different embeddings)
Deployment: On-premise local machine (CPU-based inference)
